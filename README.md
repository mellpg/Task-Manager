# 🚀 Gerenciador de Tarefas Futurista

Este é um Gerenciador de Tarefas (To-Do List) com uma estética **tecnológica e animada**, desenvolvido como parte do meu portfólio de desenvolvimento web. O projeto foca em uma experiência de usuário (UX) imersiva, utilizando elementos de design moderno e persistência de dados real.

## 🛠️ Funcionalidades

* **Gestão Completa:** Adição de tarefas, marcação de conclusão e cancelamento.
* **Interatividade Visual:** Ao mudar o status de uma tarefa (Feito ou Cancelado), a interface reage visualmente com mudanças de cores e estilos, como fundo verde para feitos e vermelho para cancelados.
* **Persistência de Dados:** Integração com o navegador para que as tarefas não sejam perdidas ao fechar ou atualizar a página.
* **Design Responsivo e Moderno:** Uso de fontes geométricas (*Orbitron* e *Inter*), efeitos de brilho neon e animações de entrada (*fade-slide-in*).
* **Limpeza em Massa:** Opção para apagar todas as tarefas da lista de uma vez através de um botão dedicado.

## 💻 Tecnologias Utilizadas

* **HTML5:** Estruturação semântica de múltiplas páginas, incluindo Home, Sobre Mim, Projetos e Contato.
* **CSS3:** Estilização avançada com `flexbox`, animações de `@keyframes` para um efeito "vivo" e uso de variáveis para estética neon.
* **JavaScript (ES6+):** Lógica de manipulação do DOM e gerenciamento de estado da aplicação.

## 🧠 Aprendizados Técnicos

Neste projeto, foquei em consolidar conhecimentos fundamentais de Front-end:

1.  **Manipulação de JSON:** Aprendi a converter objetos JavaScript em strings e vice-versa para trafegar dados de forma estruturada entre a aplicação e o armazenamento.
2.  **Web Storage API (LocalStorage):** Implementei a lógica de `setItem` e `getItem` para garantir que o usuário tenha seus dados salvos localmente.
3.  **Lógica de Arrays:** Gerenciamento de listas dinâmicas, utilizando métodos como `forEach`, `push` e manipulação por índices para atualizar o status das tarefas.
